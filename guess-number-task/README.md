### Описание проекта    
Угадать загаданное компьютером число за минимальное число попыток.

:arrow_up:[к оглавлению](_)


### Какой кейс решаем?    
Нужно написать программу, которая угадывает число за минимальное число попыток

**Условия соревнования:**  
- Компьютер загадывает целое число от 0 до 100, и нам его нужно угадать. Под «угадать», подразумевается «написать программу, которая угадывает число».
- Алгоритм учитывает информацию о том, больше ли случайное число или меньше нужного нам.

**Метрика качества**     
Результаты оцениваются по среднему количеству попыток при 1000 повторений

**Что практикуем**     
Учимся писать хороший код на python


### Краткая информация о данных
Для тестирования алгоритма используется массив из 1000 случайных чисел в диапазоне от 1 до 100. 
Использование (np.random.seed(1)) обеспечивает воспроизводимость результатов.

### Результаты:  
-Базовый алгоритм (game_v2.py): 101 попытка в среднем
-Улучшенный алгоритм (game_v3.py): 15-20 попыток в среднем
-Цель достигнута: Менее 20 попыток 

:arrow_up:[к оглавлению](.README.md#Оглавление)


### Выводы:  
1. Алгоритм с сужающимся диапазоном эффективнее случайных догадок в 5-6 раз
2. Ключевая идея — использование информации о предыдущих попытках
3. Среднее количество попыток ~18 удовлетворяет условиям задачи
4. Алгоритм стабилен: минимально 5-7, максимально 30-40 попыток
5. Фиксация random seed обеспечивает воспроизводимость результатов
